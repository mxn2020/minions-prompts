Thank you for your interest in contributing to Minions Prompts!

## Getting Started

1. Fork the repository on GitHub
2. Clone your fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/minions-prompts.git
   ```
3. Install dependencies:
   ```bash
   pnpm install
   ```
4. Create a feature branch:
   ```bash
   git checkout -b feature/my-feature
   ```

## Development Commands

<Tabs syncKey="lang">
  <TabItem label="TypeScript">
  ```bash
  # Build all packages
  pnpm run build

  # Run tests
  pnpm run test

  # Type check / lint
  pnpm run lint
  ```
  </TabItem>
  <TabItem label="Python">
  ```bash
  # Install the Python SDK in development mode
  cd packages/python
  pip install -e ".[test]"

  # Run Python tests
  pytest -v

  # Run a single test file
  pytest tests/test_renderer.py -v
  ```
  </TabItem>
</Tabs>

## Project Structure

| Directory | Description |
|-----------|-------------|
| `packages/core/` | TypeScript SDK (`@minions-prompts/sdk` on npm) |
| `packages/python/` | Python SDK (`minions-prompts` on PyPI) |
| `packages/cli/` | CLI tool (`@minions-prompts/cli`) |
| `apps/docs/` | Astro Starlight documentation site |
| `apps/web/` | Playground web app |
| `spec/` | Specification documents |
| `examples/` | Usage examples |

## Pull Request Process

1. Ensure your code passes `pnpm run lint` and `pnpm run test`
2. For Python changes, ensure `pytest -v` passes in `packages/python/`
3. Update documentation if your changes affect the public API
4. Add tests for new functionality
5. Keep PRs focused â€” one feature or fix per PR

## Code Style

- **TypeScript strict mode** for all TS packages
- **Python 3.11+** with type annotations for the Python SDK
- **ESM modules** (`"type": "module"`)
- **JSDoc/docstrings** on all public exports
