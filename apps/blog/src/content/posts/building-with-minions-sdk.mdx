# Building a Prompt Playground

One of the best ways to iterate on prompts is to see them render in real time as you edit. In this tutorial, we'll walk through building an interactive prompt playground using the Minions Prompts SDK.

## Prerequisites

- Node.js 20+
- The `@minions-prompts/sdk` package

```bash
npm install @minions-prompts/sdk
```

## Creating a Template

Start by defining a prompt template with variables:

```typescript
import { PromptTemplate } from '@minions-prompts/sdk';

const template = new PromptTemplate({
  name: 'summarizer',
  content: `Summarize the following {{content_type}} in {{word_count}} words or fewer.
Focus on: {{focus_areas}}.
Audience: {{audience}}.

Content:
{{content}}`,
  variables: {
    content_type: { type: 'string', default: 'article' },
    word_count: { type: 'number', default: 100 },
    focus_areas: { type: 'string', default: 'key takeaways' },
    audience: { type: 'string', default: 'general' },
    content: { type: 'string', required: true },
  },
});
```

## Rendering with Variables

The SDK handles variable interpolation and validation:

```typescript
const rendered = template.render({
  content_type: 'research paper',
  word_count: 150,
  focus_areas: 'methodology and findings',
  audience: 'PhD researchers',
  content: '...',
});

console.log(rendered);
// "Summarize the following research paper in 150 words or fewer..."
```

<Callout type="info">
The SDK validates all variables at render time. Missing required variables throw descriptive errors, and type mismatches are caught before the prompt reaches your LLM.
</Callout>

## Version History

Every time you save a template, a new version is created. You can traverse the history and diff any two versions:

```typescript
// Create a new version
const v2 = template.fork({
  content: `Summarize the following {{content_type}} in exactly {{word_count}} words.
Be precise and factual. Audience: {{audience}}.

Content:
{{content}}`,
});

// Diff the two versions
const diff = template.diff(v2);
console.log(`${diff.additions} lines added, ${diff.deletions} lines removed`);
```

## Running Tests

Write test cases to validate that your prompt produces the expected results:

```typescript
const results = await template.runTests([
  {
    variables: { content: 'Sample text...', word_count: 50 },
    assertions: [
      { type: 'max_tokens', value: 75 },
      { type: 'no_hallucination', reference: 'Sample text...' },
    ],
  },
]);

console.log(`${results.passed}/${results.total} tests passed`);
```

<Callout type="tip">
Integrate `template.runTests()` into your CI pipeline to catch prompt regressions before they reach production.
</Callout>

## What You Can Build From Here

This basic playground can be extended with:

- **Side-by-side diff viewer** — Compare two prompt versions visually
- **A/B test launcher** — Run the same input through two template versions and compare outputs
- **Scoring dashboard** — Track prompt quality scores over time

Check out the full playground implementation in the [web app source code](https://github.com/mxn2020/minions-prompts) and the [SDK documentation](https://prompts.minions.help).
