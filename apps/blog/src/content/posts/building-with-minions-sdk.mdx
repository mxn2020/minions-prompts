# Getting Started with the Prompts SDK

In our previous post, we introduced why prompt engineering needs proper version control. Today, we're diving into the code. The `@minions-prompts/sdk` gives you a type-safe, structured approach to managing prompts at scale.

## Installation

```bash
pnpm add @minions-prompts/sdk
```

<Callout type="warning">
The SDK requires `minions-sdk` as a peer dependency. Make sure it's installed in your project.
</Callout>

## Creating Prompt Templates

Templates are the core unit in Minions Prompts. They define a prompt's structure, variables, and metadata.

```typescript
import { createMinionsPrompts } from '@minions-prompts/sdk';

const client = createMinionsPrompts();

// Create a template with variable interpolation
const template = await client.templates.create({
  name: 'code-reviewer',
  content: `Review the following {{language}} code for {{criteria}}:

\`\`\`{{language}}
{{code}}
\`\`\`

Provide specific, actionable feedback.`,
  variables: ['language', 'code', 'criteria'],
  tags: ['code-review', 'development']
});
```

## Variable Interpolation

The rendering engine handles variable substitution with type safety.

```typescript
import { PromptRenderer } from '@minions-prompts/sdk';

const renderer = new PromptRenderer();

const rendered = renderer.render(template, {
  language: 'TypeScript',
  code: 'const x: any = "hello";',
  criteria: 'type safety and best practices'
});

console.log(rendered);
// Review the following TypeScript code for type safety and best practices:
// ```TypeScript
// const x: any = "hello";
// ```
// Provide specific, actionable feedback.
```

## Version Chains

As you iterate on prompts, each version forms a chain — making it easy to track evolution and roll back.

```typescript
// Original version (v1)
const v1 = template;

// Create v2 with more specific instructions
const v2 = await client.templates.createVersion(template.id, {
  content: `You are a senior {{language}} developer. Review the code below for {{criteria}}.
Rate each issue as LOW, MEDIUM, or HIGH severity.

\`\`\`{{language}}
{{code}}
\`\`\``,
  changelog: 'Added severity ratings and senior developer persona'
});

// View version history
const history = await client.templates.getVersionChain(template.id);
console.log(`${history.length} versions of "${template.name}"`);
```

## Writing Test Cases

Prompts should be tested like code. Define expected behaviors and validate them.

```typescript
const tests = await client.tests.create({
  templateId: template.id,
  cases: [
    {
      name: 'catches-any-type',
      input: {
        language: 'TypeScript',
        code: 'const x: any = "hello";',
        criteria: 'type safety'
      },
      assertions: [
        { type: 'contains', value: 'any' },
        { type: 'minLength', value: 50 }
      ]
    },
    {
      name: 'handles-python',
      input: {
        language: 'Python',
        code: 'def foo(x): return x + 1',
        criteria: 'type hints'
      },
      assertions: [
        { type: 'contains', value: 'type' }
      ]
    }
  ]
});
```

<Callout type="tip">
Start with simple structural assertions. As you build confidence, add LLM-judged evaluation criteria for semantic checking.
</Callout>

## Scoring and A/B Testing

Compare versions quantitatively to make data-driven prompt decisions.

```typescript
import { PromptScorer } from '@minions-prompts/sdk';

const scorer = new PromptScorer();

const results = await scorer.compare({
  templateId: template.id,
  versionA: v1.version,
  versionB: v2.version,
  testCases: tests.cases,
  metrics: ['accuracy', 'specificity', 'length']
});

console.log(`v1: ${results.scoreA.toFixed(2)} | v2: ${results.scoreB.toFixed(2)}`);
console.log(`Recommendation: ${results.recommendation}`);
```

## Using the CLI

For quick prompt management, use the CLI.

```bash
# Create a new template
prompts create --name summarizer --file ./prompts/summarizer.txt

# List all templates
prompts list

# Create a new version
prompts version create --template summarizer --file ./prompts/summarizer-v2.txt

# Run tests
prompts test run --template summarizer
```
