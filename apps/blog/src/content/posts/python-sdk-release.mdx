# Python SDK for Minions Prompts

We're thrilled to announce the release of the **Minions Prompts Python SDK** — bringing version-controlled prompt engineering to the Python ecosystem, where most AI/ML development happens.

<Callout type="info">
The Python SDK provides full feature parity with the TypeScript SDK, including template management, version chains, variable interpolation, and test-driven prompt development.
</Callout>

## Installation

```bash
pip install minions-prompts
```

## Quick Start

```python
from minions_prompts import PromptTemplate

template = PromptTemplate(
    name="code-reviewer",
    content="Review the following {{language}} code for {{focus}}:\n\n{{code}}",
    variables={
        "language": {"type": "string", "default": "Python"},
        "focus": {"type": "string", "default": "bugs and performance"},
        "code": {"type": "string", "required": True},
    },
)

rendered = template.render(
    language="Python",
    focus="security vulnerabilities",
    code="user_input = input(); exec(user_input)",
)
print(rendered)
```

## Jupyter Notebook Integration

The Python SDK is designed to work seamlessly in Jupyter notebooks, making it perfect for interactive prompt iteration:

```python
# In a notebook cell
from minions_prompts import PromptTemplate

template = PromptTemplate.load("summarizer")

# Iterate on the prompt
v2 = template.fork(
    content="Provide a concise summary of {{content}} in {{style}} style."
)

# Compare outputs side by side
comparison = template.compare(v2, input={"content": "...", "style": "academic"})
comparison.display()  # Rich output in Jupyter
```

## Test-Driven Prompt Development

Write tests for your prompts in familiar pytest style:

```python
import pytest
from minions_prompts import PromptTemplate

template = PromptTemplate.load("customer-support")

def test_professional_tone():
    result = template.render(issue="refund request", tone="professional")
    assert "refund" in result.lower()
    assert len(result.split()) < 200

def test_required_variables():
    with pytest.raises(ValueError, match="required"):
        template.render()  # missing 'issue'
```

<Callout type="tip">
Add prompt tests to your CI pipeline with `pytest tests/prompts/` — catch regressions every time a template is modified.
</Callout>

## LangChain & LlamaIndex Integration

The Python SDK exports prompts in formats compatible with popular frameworks:

```python
from minions_prompts import PromptTemplate

template = PromptTemplate.load("qa-assistant")

# Export to LangChain format
langchain_prompt = template.to_langchain()

# Export to LlamaIndex format
llama_prompt = template.to_llama_index()
```

## Language Interoperability

Both the TypeScript and Python SDKs use the same storage format and version chain protocol. Create a template with the TypeScript CLI, iterate on it in a Jupyter notebook, and deploy it from your Node.js server — everything stays in sync.
